# CI/CD Integration Guide

This document explains how to use AgentShield in continuous integration pipelines.

## Workflows shipped

Two GitHub Actions workflow files live in `.github/workflows/`:

```
.github/workflows/
  ci.yml        Lint (ruff) + unit tests (pytest)
  scan.yml      Security scan, report artifacts, PR comment, fail-on gate
```

## `ci.yml` — Lint and test

Runs on every push and pull request to `main` / `master`.

```
Steps:
  1. Checkout
  2. Set up Python 3.11
  3. pip install -e ".[dev]"
  4. ruff check agentshield tests
  5. pytest --tb=short -q
```

No configuration required. All 49 unit tests must pass and the linter must
produce zero errors for the job to succeed.

## `scan.yml` — Security scan

### Triggers

- Push to `main` or `master`
- Pull request targeting `main` or `master`
- Manual (`workflow_dispatch`) with optional inputs

### Manual inputs (workflow_dispatch)

| Input | Default | Description |
|---|---|---|
| `scan_path` | `.` | Relative path inside the repo to scan |
| `fail_on` | `high` | Minimum severity that fails the workflow |

### What the workflow does

1. Installs AgentShield from source using `pip install -e ".[dev]"`.
2. Runs `agentshield scan <scan_path> --format both --output agentshield-reports --fail-on <fail_on>`.
3. Captures the exit code without immediately failing so reports are always uploaded.
4. Uploads `findings.json` as artifact `agentshield-findings-json`.
5. Uploads `findings.md` as artifact `agentshield-findings-md`.
6. On pull requests, generates a compact `pr-summary.md` and posts it as a PR comment using `GITHUB_TOKEN`.
7. If the scan exit code was `1` (findings at or above threshold), the workflow fails with an explanatory message.

### Exit code semantics

| Exit code | Meaning |
|---|---|
| 0 | Scan clean — no findings at or above `--fail-on` severity |
| 1 | One or more findings at or above `--fail-on` severity |
| 2 | Invalid CLI argument (e.g. unknown `--fail-on` value) |

### Adjusting the failure threshold

To allow high findings but fail only on critical:

```yaml
- name: Run scan
  run: |
    python -m agentshield scan . \
      --format both \
      --output agentshield-reports \
      --fail-on critical
```

To enforce stricter gating (fail on medium or higher):

```yaml
      --fail-on medium
```

### Required permissions

The workflow declares:

```yaml
permissions:
  contents: read
  pull-requests: write
```

`pull-requests: write` is only exercised during PR events (the comment step is
gated on `if: github.event_name == 'pull_request'`). Remove it if you do not
want PR comments.

## PR comment format

On every pull request, the workflow posts a comment like:

```markdown
## AgentShield scan 🔴 **FAILED**

| Metric | Value |
| --- | --- |
| Findings | 6 |
| High / Critical | 3 |
| Risk score | 54 / 100 |
| Target | `.` |

**High / Critical findings:**

- 🟠 `HIGH` — Suspicious imperative instruction: 'do not tell the user'
- 🔴 `CRITICAL` — Strong exfiltration signal: 'api key'

_Full report available as a workflow artifact (`agentshield-findings-md`)._
```

The comment is generated by `agentshield.reporting.markdown_report.pr_comment_body()`.

## Running the full CI flow locally

This mirrors exactly what the two GitHub Actions workflows do:

```bash
# 1. Install
pip install -e ".[dev]"

# 2. Lint
ruff check agentshield tests

# 3. Test
pytest --tb=short -q

# 4. Security scan
python -m agentshield scan . \
  --format both \
  --output agentshield-reports \
  --db-path agentshield.db \
  --fail-on high \
  --verbose

echo "Exit code: $?"
```

## Integrating with other CI systems

AgentShield is a standard Python CLI tool. Any CI system that can run Python
commands can use it.

**CircleCI:**

```yaml
- run:
    name: AgentShield security scan
    command: |
      pip install -e ".[dev]"
      python -m agentshield scan . --format json --output reports --fail-on high
- store_artifacts:
    path: reports/findings.json
```

**GitLab CI:**

```yaml
agentshield:
  image: python:3.11
  script:
    - pip install -e ".[dev]"
    - python -m agentshield scan . --format both --output reports --fail-on high
  artifacts:
    paths:
      - reports/
    when: always
```

**Pre-commit hook (local gate):**

```bash
# .git/hooks/pre-commit
#!/usr/bin/env bash
python -m agentshield scan . --format json --output /tmp/as-pre-commit --fail-on high --verbose
```

## Artifacts

After a scan workflow run, two artifacts are available for download from the
Actions run page:

| Artifact name | Contents |
|---|---|
| `agentshield-findings-json` | `findings.json` — machine-readable full report |
| `agentshield-findings-md` | `findings.md` — human-readable Markdown report |

The JSON report can be consumed by downstream tooling (e.g. SIEM ingestion,
ticket creation, dashboard feeds) without re-running the scan.
